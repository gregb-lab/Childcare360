# Patch: RosteringModule.jsx — Grid day tab missing first day

## Bug
When a roster period starts on a non-Monday (e.g. Sunday 22 Mar), the grid
day tabs skip to Monday, dropping the first day from the view. The period
header shows 22→26 Mar but tabs show Mon 23, Tue 24, Wed 25, Thu 26.

## Root cause
The grid day array is generated by snapping to "Monday of the week" rather
than iterating from period.start_date to period.end_date.

## Search for this pattern in RosteringModule.jsx:
Look for any of these patterns near the Grid tab render:
  - getDay() === 1  or  .getDay() - 1
  - "Monday"  alongside  start_date
  - Array of 5 dates built from a date adjusted by getDay()

## Fix — replace the day generation with:

```jsx
// BEFORE (typical buggy pattern):
const getWeekDays = (startDate) => {
  const d = new Date(startDate + 'T12:00:00');
  const monday = new Date(d);
  monday.setDate(d.getDate() - (d.getDay() === 0 ? 6 : d.getDay() - 1));
  return Array.from({ length: 5 }, (_, i) => {
    const day = new Date(monday);
    day.setDate(monday.getDate() + i);
    return day.toISOString().slice(0, 10);
  });
};

// AFTER (correct — iterate from actual start_date to end_date):
const getGridDays = (startDate, endDate) => {
  const days = [];
  const end = new Date(endDate + 'T12:00:00');
  let cur = new Date(startDate + 'T12:00:00');
  while (cur <= end) {
    days.push(cur.toISOString().slice(0, 10));
    cur = new Date(cur);
    cur.setDate(cur.getDate() + 1);
  }
  return days;
};
```

Then wherever the grid days are used, call:
  `getGridDays(selectedPeriod.start_date, selectedPeriod.end_date)`
instead of:
  `getWeekDays(selectedPeriod.start_date)`

## Also: update the day-tab label to include the day name:
```jsx
const fmt = (dateStr) => {
  const d = new Date(dateStr + 'T12:00:00');
  const dayNames = ['Sun','Mon','Tue','Wed','Thu','Fri','Sat'];
  return `${dayNames[d.getDay()]} ${d.getDate()}`;
};
```
